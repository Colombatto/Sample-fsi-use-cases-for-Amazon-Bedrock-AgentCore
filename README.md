## [UC1 — Real-Time Fraud Investigation Agent](01_fraud-investigation-agent/README.md)

### The Problem

Financial institutions process millions of transactions every day. When a potentially fraudulent transaction occurs, the window to act is narrow — seconds to minutes. Traditional fraud detection relies on rules engines that flag transactions, but the investigation and response process is largely manual: an analyst reviews the alert, looks up transaction history, cross-references known fraud patterns, decides whether to freeze the card, contacts the customer, and opens a case. This process is slow, inconsistent, and doesn't scale.

The challenge is not just detection — it's the end-to-end response: from the moment an anomaly is spotted to the moment the customer is protected and the case is documented, with a full audit trail that satisfies regulatory requirements.

### The Solution

A multi-agent system triggered by either an automated rule (e.g. a transaction exceeds a threshold or originates from a high-risk country) or a direct human report (e.g. a customer flags a suspicious charge via the bank's app). From that point, the system autonomously retrieves and analyses the customer's transaction history, cross-references known fraud patterns to compute a risk score, and takes action based on the risk level.

High-risk cases are handled automatically — the card is frozen, the customer is notified, and the case is recorded with a full audit trail. Medium-risk cases pause for human approval: the agent notifies a fraud analyst with a case summary and a dashboard link, then waits — without blocking any resources — until the analyst approves or rejects the action. This async pause/resume pattern is one of AgentCore's key differentiators.

Every step is traced via OpenTelemetry to CloudWatch, producing a complete audit trail for regulatory compliance (PCI-DSS, GDPR).

**AgentCore capabilities:** Runtime (multi-agent, async pause/resume), Memory (short and long-term), Gateway (MCP tools for APIs and notifications), Policy (Cedar rules for approval thresholds), Identity (agent auth to downstream systems), Observability (full audit trail)

## [UC2 — Autonomous AML / Compliance Monitoring Agent](02_aml-compliance-agent/README.md)

### The Problem

Anti-Money Laundering (AML) compliance is one of the most resource-intensive operations in financial services. Compliance teams are required to continuously monitor transactions, identify suspicious activity patterns, research entities against sanctions lists, and file Suspicious Activity Reports (SARs) with regulators — all within strict deadlines. The volume of data involved is enormous: a mid-size bank may process hundreds of thousands of transactions per day, and the number of false positives generated by traditional rule-based systems is high, leading to analyst fatigue and missed genuine threats.

Manual investigation is slow, expensive, and error-prone. Regulatory penalties for AML failures are severe — both financially and reputationally. The challenge is to monitor continuously, investigate thoroughly, and document everything — at a scale that human teams alone cannot sustain.

### The Solution

An autonomous AML monitoring agent that runs continuously as a long-running async workload, processing transaction streams, correlating suspicious activity across accounts and time windows, and researching entities against public sanctions lists and regulatory databases. When a suspicious pattern is identified, the agent drafts a structured investigation report and — where thresholds are met — initiates the SAR filing process.

The agent uses AgentCore's extended runtime (up to 8 hours) to handle the long-running nature of AML investigations, which often require correlating activity across days or weeks of transaction history. A Code Interpreter sub-agent runs statistical analysis on transaction graphs to identify layering and structuring patterns that simple rules would miss. A Browser agent scrapes public sanctions lists (OFAC, UN, EU), regulatory bulletins, and company registries to enrich entity profiles in real time.

Human oversight is built in: the agent flags cases for compliance officer review before any SAR is filed, ensuring regulatory accountability is maintained. All findings are stored with a complete evidence trail.

**AgentCore capabilities:** Runtime (long-running async, up to 8h), Browser (sanctions lists, regulatory sources), Code Interpreter (statistical graph analysis), Gateway (MCP tools for internal case management, regulatory APIs), Policy (Cedar rules for SAR filing thresholds and human approval gates), Observability (evidence trail for regulatory defensibility)

## UC3 — Personalised Wealth Management Advisor

### The Problem

Personalised investment advice has traditionally been the preserve of high-net-worth clients who can afford dedicated relationship managers. For the mass affluent segment, the experience is typically generic: standardised portfolios, infrequent reviews, and advice that doesn't account for the client's full financial picture — tax situation, life events, risk tolerance changes, or market conditions.

At the same time, human advisors are constrained by time: they can only manage a limited number of client relationships deeply. Scaling personalised advice without scaling headcount is a fundamental challenge for wealth management firms. The risk of giving advice that is inconsistent, outdated, or not aligned with the client's actual situation is both a compliance risk and a client retention risk.

### The Solution

A multi-agent wealth management system where specialised agents collaborate to generate personalised, contextually aware investment recommendations. A Portfolio Agent analyses the client's current holdings and performance. A Market Research Agent monitors relevant market data, news, and economic indicators. A Tax Optimisation Agent evaluates the tax implications of proposed changes. An Orchestrator Agent synthesises the outputs of all three and generates a recommendation tailored to the client's risk profile, goals, and constraints.

The system uses AgentCore Memory to maintain long-term context about each client — their stated preferences, past advice, life events, and how their portfolio has evolved — so every interaction builds on prior knowledge rather than starting from scratch. Short-term memory maintains the full context of the current advisory session.

AgentCore Identity handles OAuth authentication with the client's brokerage account, ensuring the agent accesses only the data it is authorised to see. A Code Interpreter sub-agent runs portfolio simulations and Monte Carlo analysis to quantify risk and project outcomes. Before any recommendation is surfaced to the client or advisor, AgentCore Evaluations scores the output for quality and consistency, acting as an automated quality gate.

**AgentCore capabilities:** Runtime (multi-agent orchestration), Memory (long-term client profile and advice history, short-term session context), Gateway (market data APIs, portfolio management systems, tax calculation services), Identity (OAuth with brokerage accounts via Cognito/Okta), Code Interpreter (portfolio simulations, Monte Carlo analysis), Evaluations (recommendation quality scoring before surfacing to advisors)

## UC4 — Intelligent Loan Underwriting Agent

### The Problem

Loan underwriting is a complex, document-heavy process that typically takes days to weeks. An underwriter must collect and verify multiple data sources — credit bureau reports, bank statements, payslips, employment verification, property valuations — synthesise them into a risk assessment, and produce a structured decision with supporting rationale. The process is largely manual, inconsistent across underwriters, and slow — creating friction for applicants and operational cost for lenders.

Regulatory requirements add further complexity: underwriting decisions must be explainable and defensible under fair lending laws (ECOA, FCRA in the US; equivalent regulations in other markets). A decision that cannot be explained is a compliance liability. At the same time, the pressure to reduce time-to-decision is intense, as applicants increasingly expect near-instant responses.

### The Solution

An end-to-end loan underwriting agent that automates the full assessment lifecycle — from application intake to structured decision output — while maintaining a complete, explainable audit trail for every decision.

The agent uses AgentCore Gateway to connect to credit bureaus, payroll verification APIs, and document verification services as MCP tools, pulling all required data sources in parallel rather than sequentially. A Code Interpreter sub-agent runs credit scoring models, debt-to-income calculations, and stress tests against interest rate scenarios. A Browser agent verifies employer details against public records and company registries where API access is not available.

Each application runs in its own isolated session via AgentCore Runtime, ensuring no data leakage between concurrent applications. AgentCore Memory retains prior application history for returning customers, allowing the agent to identify patterns (e.g. repeated applications, previous defaults) that inform the current assessment.

AgentCore Policy enforces decisioning guardrails: applications above a defined exposure threshold require human underwriter review before a decision is issued. The full reasoning chain — every data point retrieved, every calculation performed, every rule applied — is captured via AgentCore Observability, producing the explainability trail required for fair lending compliance.

**AgentCore capabilities:** Runtime (session isolation per application, multi-agent), Gateway (credit bureaus, payroll APIs, document verification as MCP tools), Code Interpreter (credit scoring, DTI calculations, stress tests), Browser (employer verification, public records), Memory (returning customer history), Policy (exposure thresholds and human review gates), Observability (explainability trail for ECOA/FCRA compliance)

## UC5 — Regulatory Change Management Agent

### The Problem

Financial institutions operate under a constantly evolving regulatory landscape. New rules and amendments are published continuously by bodies such as the SEC, FINRA, FCA, EBA, and the Basel Committee — across multiple jurisdictions simultaneously. Compliance teams must monitor these publications, assess which internal policies, processes, and systems are impacted, perform gap analyses, and drive remediation — all within regulatory deadlines that can be as short as 90 days.

The volume of regulatory text is enormous and growing. A single regulatory update can run to hundreds of pages. Mapping its requirements to internal controls, policies, and technology systems requires deep institutional knowledge and significant analyst time. Most firms rely on manual processes supported by spreadsheets and email — a fragile, slow, and error-prone approach that creates regulatory risk and diverts skilled compliance staff from higher-value work.

### The Solution

An autonomous regulatory change management agent that monitors regulatory publications across multiple jurisdictions, identifies impacted internal policies and systems, drafts structured gap analyses, and creates remediation tasks — running as a long-running background workload that operates continuously without human initiation.

The agent uses AgentCore Browser to monitor regulatory body websites (SEC EDGAR, FCA Handbook, EUR-Lex, Federal Register), detect new publications, and extract relevant content. When a new regulation or amendment is identified, the agent retrieves the full text, parses it using a Code Interpreter sub-agent, and compares it against the institution's internal policy library to identify gaps and conflicts.

AgentCore Memory maintains a long-term knowledge base of regulations already processed, mapped controls, and prior gap analyses — so the agent builds institutional knowledge over time rather than starting from scratch with each update. When a gap is identified, the agent drafts a structured impact assessment and pushes it to the firm's GRC system or project management tool via AgentCore Gateway (JIRA, Confluence, ServiceNow).

Human oversight is maintained at the output stage: all gap analyses and remediation tasks are routed to the relevant compliance owner for review and sign-off before being actioned. AgentCore Evaluations scores the quality of each gap analysis before it is surfaced, reducing noise and ensuring only well-formed outputs reach the compliance team.

**AgentCore capabilities:** Runtime (long-running async background workload), Browser (regulatory body websites, Federal Register, EUR-Lex), Code Interpreter (regulatory text parsing, policy diff analysis), Memory (long-term regulatory knowledge base, processed regulation history), Gateway (GRC systems, JIRA, Confluence, ServiceNow as MCP tools), Evaluations (gap analysis quality scoring before routing to compliance owners), Observability (full processing trail for regulatory defensibility)
